## Project Overview

**KernelFlow** — An automated CUDA kernel optimization and CI/CD deployment platform for LLM inference.

The core idea: a researcher pushes a fused CUDA kernel to GitHub, the system automatically builds, tests, benchmarks, and deploys it as a PyTorch Extension. The platform is designed to mirror real production MLOps infrastructure (NVIDIA-style), running locally on a GPU PC with a MacBook as the development/monitoring machine.

This project is directly aligned with NVIDIA's Deep Learning Libraries Group (cuDNN / FlashInfer infrastructure team) job requirements.

---

## Project Goals

1. Implement fused CUDA kernels that improve LLM inference performance
2. Build a fully automated CI/CD pipeline around those kernels
3. Simulate a cloud Kubernetes deployment environment locally at zero cost

---

## Fused Kernels to Implement (Milestones)

| Milestone | Kernel | Why it matters |
|-----------|--------|---------------|
| 1 | Fused RMSNorm + RoPE | Every LLaMA Transformer block starts with these two ops — fusing saves one full HBM round-trip |
| 2 | Fused SiLU + Elementwise Multiply | LLaMA FFN gate — two ops become one kernel |
| 3 | Fused Attention (simplified Flash Attention) | Eliminates intermediate softmax(QK^T) write to HBM |

Each kernel must beat its unfused baseline by a defined speedup gate before it can be deployed.

## Full Tech Stack

### Kernel / Compute Layer
- **CUDA C++** — fused kernel implementation
- **cuBLAS** — matmul baseline for comparison
- **cuDNN** — activation/conv reference implementations
- **Triton** — Python reference implementation for numerical correctness validation
- **PyTorch Extension** (pybind11 + setup.py) — exposes kernels to Python

### Build Layer
- **CMake** — multi-kernel build system, links cuBLAS/cuDNN
- **Make** — generated by CMake, called by Jenkins
- **Docker** — unified build environment (`nvidia/cuda:12.0-devel`)
- **Bazel** — optional alternative build target (for JD alignment)

### CI/CD Layer
- **GitHub** — source control, PR webhooks
- **Jenkins** — pipeline orchestration (Blue Ocean UI), hosted on GPU PC
- **Kubernetes (minikube)** — isolates each pipeline job in a pod
- **NVIDIA Device Plugin** — enables GPU access inside K8s pods

### Quality Layer
- **clang-tidy** — C++ static analysis
- **compute-sanitizer** — GPU memory safety (out-of-bounds, race conditions)
- **ruff / pylint** — Python static analysis
- **pytest + lcov** — test coverage
- **Codecov** — coverage trend visualization

### Visualization Layer
- **Nsight Systems** — kernel timeline, CPU/GPU sync analysis
- **Nsight Compute** — roofline analysis, memory bandwidth utilization
- **wandb** — benchmark history tracking across commits
- **Grafana + Prometheus** — system-level monitoring dashboard
- **Jenkins Blue Ocean** — pipeline status visualization

## Jenkins Pipeline Stages

```
Build → Static Analysis → Test + Coverage → Benchmark → Deploy
```

- **Static Analysis** runs clang-tidy, compute-sanitizer, ruff in parallel
- **Benchmark gate**: kernel must exceed minimum speedup vs baseline or pipeline fails
- **Deploy** only triggers on `main` branch — packages as wheel, pushes to local registry

### Benchmark Gates

| Kernel | Min Speedup | Max Numerical Error |
|--------|-------------|-------------------|
| fused_rmsnorm_rope | 1.5x | 1e-5 |
| fused_silu_mul | 1.3x | 1e-5 |
| fused_attention | 2.0x | 1e-4 |

## Build & Test Commands

```bash
# Build
cmake -B build -DENABLE_COVERAGE=ON
make -C build -j$(nproc)

# Test
pytest tests/ --cov=. --cov-report=xml -v

# Benchmark
./build/bench_all
python benchmarks/report.py

# Static analysis
clang-tidy kernels/**/*.cu -- -I/usr/local/cuda/include
compute-sanitizer --tool memcheck ./build/test_kernel
ruff check tests/ benchmarks/

# Package
python setup.py bdist_wheel
```
